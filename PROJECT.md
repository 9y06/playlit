# 프로젝트 아이디어 문서

> 2025년 1학기 웹 서비스 개발 — Next.js 수행평가

---

## 1. 프로젝트 이름 또는 가칭

**Playlit** — 플랫폼 상관없이 플레이리스트를 만들고, 이전하고, 공유하는 서비스

---

## 2. 프로젝트 개요

Playlit은 스포티파이, 유튜브뮤직, 애플뮤직 등 플랫폼이 달라도 내 플레이리스트를 한 곳에 모으고 다른 사람과 공유할 수 있는 크로스 플랫폼 플레이리스트 서비스입니다.

기존 플랫폼 링크를 붙여넣으면 곡 목록을 자동으로 가져와 Playlit 안에 저장(이전)할 수 있고, Playlit 안에서 직접 곡을 검색해 새 플레이리스트를 만들 수도 있습니다. 무드·상황 태그를 달아 분류하고, 다른 사람의 플레이리스트를 탐색하거나 내 컬렉션에 저장할 수 있습니다. 한국어·영어 다국어를 지원하며 Vercel에 배포합니다.

---

## 3. 주 고객

음악 플랫폼을 여러 개 쓰거나, 친구와 플랫폼이 달라서 플레이리스트 공유가 불편했던 10~20대. 기존 플랫폼에서 만든 플레이리스트를 한 곳에 모아두거나, 플랫폼 구독을 바꿀 때 플레이리스트를 이전하고 싶은 사람.

---

## 4. 주요 기능

| 기능 | 설명 |
|------|------|
| 플레이리스트 직접 만들기 | 곡 검색 후 직접 추가해 새 플레이리스트 생성 (Create) |
| 플레이리스트 이전 | 스포티파이·유튜브뮤직·애플뮤직 링크 붙여넣으면 곡 목록 자동 가져와 저장 (Create) |
| 태그 & 무드 수정 | "새벽감성", "운동할때", "시험기간" 등 태그 추가·수정 (Update) |
| 플레이리스트·곡 삭제 | 내 컬렉션에서 플레이리스트·곡 삭제 (Delete) |
| 공개/비공개 설정 | 플레이리스트별 공개 범위 설정 |
| 탐색 피드 | 다른 사람의 공개 플레이리스트 탐색, 태그 필터링 |
| 저장하기 | 다른 사람 플레이리스트 내 컬렉션에 저장 |
| 로그인 & 인증 | Google OAuth (NextAuth.js), 미들웨어로 비로그인 접근 차단 |
| 다국어 지원 | 한국어·영어 전환, URL 경로 방식 (`/ko`, `/en`) |

### 화면 구성 (3개 이상, 로그인/회원가입 제외)

1. **탐색 피드** — 최신·인기 공개 플레이리스트, 태그별 필터
2. **플레이리스트 상세** — 곡 목록, 태그, 저장 버튼
3. **내 컬렉션** — 내가 만들거나 저장한 플레이리스트 관리
4. **플레이리스트 만들기** — 곡 검색 후 추가해서 새 플레이리스트 생성
5. **플레이리스트 이전** — 외부 링크 붙여넣기 → 곡 목록 미리보기 → 저장

---

## 5. 기술적 특이사항

- **크로스 플랫폼 링크 파싱**: 스포티파이(Spotify Web API), 유튜브(YouTube Data API v3), 애플뮤직(링크 파싱)을 Route Handler로 프록시 처리해 API 키 노출 방지
- **곡 검색**: Spotify Web API로 곡 검색, 플레이리스트 직접 조합 가능
- **Server Action 활용**: 플레이리스트 생성·태그 수정·삭제를 Server Action으로 처리 (revalidatePath 포함)
- **DB 연동**: Prisma + PostgreSQL (Supabase 무료 플랜)
- **인증**: NextAuth.js + Google OAuth 2.0, `middleware.ts`로 `/collection`, `/create`, `/import` 등 보호 라우트 설정
- **이미지 최적화**: `next/image`로 앨범아트 최적화, 각 플랫폼 CDN 도메인 허용 설정
- **동적 OG 메타데이터**: 플레이리스트 상세 페이지에서 첫 번째 앨범아트를 `og:image`로 자동 적용
- **로딩 & 에러 UI**: `loading.tsx` + `Suspense`, `error.tsx`, `not-found.tsx`
- **sitemap.xml** 자동 생성 + `robots.txt` 포함
- **배포**: Vercel, 환경변수로 Spotify/YouTube API 키, NextAuth Secret, DB URL 관리

### Codex(바이브코딩) 감안 포인트

Spotify API, YouTube Data API, NextAuth, Prisma, Server Action 전부 Codex가 공식 패턴 그대로 잘 뽑아주는 영역입니다. 링크 파싱 로직도 패턴이 명확해서 AI가 잘 처리합니다. 직접 손댈 부분은 각 플랫폼 API 키 `.env` 세팅과 프롬프트 방향 조정 정도입니다.
